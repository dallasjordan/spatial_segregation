Now we generate a track using traditional threshold methods from the twilight pairs, clamp the start and end, and prepare for estimation. We have two fixed points, and the start and end of the track - these won't be allowed to be estimated in the model but help constrain the overall path taken. 

xp <- threshold.path(prs$Twilight, prs$Rise, zenith = 95, unfold = TRUE)

x0[1, 1:2] <- start
x0[nrow(x0), 1:2] <- end

Create functions to test location estimates and return prior likelihoods, is.sea() simply tests that we are within a sensible longitude/latitude range and that we fall in non-land areas. This function can be expanded to include further tests such as land vs. sea and include temporal components such as SST, but we ignore that complication for now.

The track is pretty wide-ranging, so we build a mask and zap all the out-of-range coordinates and interpolate between our conservative first-guess before proceeding.

## function to generate a mask
land.mask <- function(poly, xlim, ylim, n = 4L, land = TRUE) {
    r <- raster(nrows = n * diff(ylim), ncols = n * diff(xlim), xmn = xlim[1L], 
        xmx = xlim[2L], ymn = ylim[1L], ymx = ylim[2L], crs = projection(poly))
    r <- rasterize(poly, r)
    r <- as.matrix(is.na(r))[nrow(r):1L, ]
    if (land) 
        r <- !r
    xbin <- seq(xlim[1L], xlim[2L], length = ncol(r) + 1L)
    ybin <- seq(ylim[1L], ylim[2L], length = nrow(r) + 1L)

    function(p) {
        r[cbind(.bincode(p[, 2L], ybin), .bincode(p[, 1L], xbin))]
    }
}

The first-guess points do include a bit of meandering over land that is definitely not correct, so here we zap a bit more of the track and re-interpolate. (The estimation will allow the track to wander outside of the range applied here, but it's very hard to allow it to detect that it should not cross land so we help it here. Stopping at 51 N keeps us pretty cleanly in the Pacific arc here. )

x0[x0[, 2] > 51, ] <- NA
x0[!log.prior(x0) == 0, 1:2] <- NA
x0 <- zoo::na.approx(x0)

Build the model and proposal functions and run. 

model <- threshold.model(prs$Twilight, prs$Rise,
                         twilight.model="ModifiedLogNormal",
                         alpha = c(1.2,0.8),
                         beta = c(2.5, 0.5),

                         logp.x=log.prior,
                         logp.z=log.prior,
                         x0=x0,z0=z0,fixedx=fixedx,zenith=95)


nn <- nrow(x0)
x.proposal <- mvnorm(S=diag(c(0.005,0.005)),n=nn)
z.proposal <- mvnorm(S=diag(c(0.005,0.005)),n=nn-1)



fit <- estelle.metropolis(model, x.proposal, z.proposal, iters = 100, thin = 20)
lastx <- chain.last(chain.collapse(fit$x))
lastz <- chain.last(chain.collapse(fit$z))

## ensure that all pass before proceeding
while (!(all(c(log.prior(lastx) == 0, log.prior(lastz) == 0)))) {
    fit <- estelle.metropolis(model, x.proposal, z.proposal, iters = 100, thin = 20, 
        x0 = lastx, z0 = lastz)
    lastx <- chain.last(chain.collapse(fit$x))
    lastz <- chain.last(chain.collapse(fit$z))
    xm <- location.mean(fit$x)
    plot(xm)
    plot(w1, add = TRUE)
}

Run a short set of iterations a few times to see that we get a sensible estimation.
